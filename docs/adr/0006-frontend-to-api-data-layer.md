# Frontend-to-API data layer: TanStack Query + OpenAPI codegen + branded IDs

The SPA talks to `Balance.Web` through TanStack Query for all reads and mutations, with wire types generated from OpenAPI at .NET build time (`Microsoft.Extensions.ApiDescription.Server` + `openapi-typescript`; the document emits under `artifacts/` so codegen needs no running app, and `npm run build` runs codegen before `tsc -b`). Generated types stay `string`; a thin `src/api/*.ts` layer wraps each fetch and re-brands IDs using the existing `Brand<>` pattern (brand-at-the-boundary). PATCH endpoints use `fast-json-patch` `compare(original, edited)` over an editable copy to produce RFC 6902 patches so the per-line dictionary patches from ADR-0014 work natively. Components use per-component `isPending` / `isError` guards with a shared `<Skeleton/>` + inline `<ErrorState/>` (no Suspense, no extra route-level error boundaries). `src/lib/domain.ts` becomes a pure view-model layer; `src/api/*.ts` owns the wire boundary.

Branding is laundered in the mapper because `StronglyTypedId` flattens to `string` in OpenAPI (ADR-0004's parameter-swap safety stops at the C# boundary), and an unbranded TS type would let an `AccountId` be passed where a `CounterpartyId` is expected — a one-line cast is cheaper than customizing OpenAPI emission. Hand-written wire types are rejected because they already drifted in this codebase. Auth and antiforgery are out of scope: no endpoint calls `RequireAuthorization` and `UseAntiforgery` does not fire on JSON minimal-API calls, so this layer assumes anonymous JSON access; both will be revisited when login lands.
