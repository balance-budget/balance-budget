# BankTransaction inbox exits via Categorize, Attach, or Dismiss

A `BankTransaction` leaves the **Inbox** in exactly three ways: (a) **Categorize** — a new `JournalEntry` is created via the **Categorization flow** (ADR-0013) and `BankTransaction.JournalEntryId` is set to it; (b) **Attach** — `JournalEntryId` is set to an existing self-transfer `JournalEntry` and the matching `JournalLine` flips from `Uncleared` to `Cleared`; (c) **Dismiss** — `DismissedAt` (UTC) and `DismissedReason` are set. The link is a nullable scalar FK on the `BankTransaction` side: a BT references at most one JE; a JE may be referenced by many BTs **only for self-transfers** (the same movement reported by two own-bank statements).

The **Attach predicate** drives both the Inbox hint and the Attach endpoint's validation. BT may attach to JE iff: (1) `BT.JournalEntryId IS NULL AND BT.DismissedAt IS NULL`; (2) a `JournalLine L1` on JE has `L1.AccountId == BT.BankAccount.AccountId`, `L1.ReconciliationStatus == Uncleared`, and `L1.Amount` matches `BT.Money.Amount` with signs aligned in the focal-account frame (positive BT = money in; debit on an `Asset` line, credit on a `Liability` line); (3) a different line `L2` on JE has an `Account` whose backing `BankAccount.Iban == BT.CounterpartyAccountNumber`; (4) `|BT.BookingDate − JE.Date| <= 7 calendar days`; (5) `L1.Account.CurrencyCode == BT.Money.CurrencyCode`; (6) every other line on JE is on an own-`Account` (self-transfer guard); (7) `JE.CounterpartyId IS NULL`. The hint is shown only when exactly one JE satisfies the predicate; zero or multiple suppress it.

`POST /api/bank-transactions/{id}/attach` (`{ JournalEntryId }`) flips the matched line to `Cleared`; `/detach` reverses it. The "multi-BT only when self-transfer" invariant is enforced in the service layer, not as a DB CHECK (it is unreachable through the API; a constraint would need a trigger or denormalized flag). We rejected auto-attaching at import time (it would mutate JE state as a side effect of a CSV upload, per ADR-0009) and fuzzy matching (silent wrong-attach is the worst failure mode; the manual JE-picker covers near-misses). Dismiss stays for rows with no JE to point at and survives re-import, since the `RowHash` dedup keeps the soft state.
