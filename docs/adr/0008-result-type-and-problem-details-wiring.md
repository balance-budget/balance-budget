# Service errors are returned as `Result<T>`, not thrown as `DomainException`

Service methods return `Result<T>` (or non-generic `Result` for side-effect-only operations like `DeleteAsync`). `ServiceError` is a sealed hierarchy in `Balance.Services.Contracts` — `NotFoundError(Entity, Id)`, `ConflictError(Code, Message)`, `InvariantError(Code, Message)`, `ValidationError(IReadOnlyDictionary<string, string[]>)` — each carrying a stable `Code` string that becomes the `code` extension on the wire `ProblemDetails`. The FluentValidation endpoint filter still short-circuits with `Results.ValidationProblem` before the handler; `JsonPatchExtensions.ApplyAndValidateAsync` returns `Result<T>` so PATCH chains patch-apply → service in one pipeline. Endpoints map results via `Balance.Web/Mappers/ResultExtensions.cs` (`.ToOk<T>()`, `.ToCreatedAt<T>(...)`, `.ToNoContent()`, …), each returning a typed `Results<TSuccess, NotFound<ProblemDetails>, Conflict<ProblemDetails>, UnprocessableEntity<ProblemDetails>, ValidationProblem>` union so the source generator populates OpenAPI response shapes. Save-time race backstops live in `DbContextExtensions.SaveAsync`, which catches `UniqueConstraintException` (→ `ConflictError("uniqueness_conflict", …)`) and `ReferenceConstraintException` (→ `ConflictError("referenced", …)`) via `EntityFrameworkCore.Exceptions`; pre-flight `AnyAsync` uniqueness checks stay in each service as the user-facing-message path and common case. Every other `DbUpdateException` propagates to a 500. `DomainException`, `DomainExceptionKind`, and `DomainExceptionHandler` are deleted; `AddProblemDetails()` provides the only remaining 500-for-unhandled handler.

The closed `ServiceError` hierarchy with `Code` strings is chosen over a flat `Result<T, Error>` (loses the not-found-vs-conflict distinction the `code` extension restores), per-operation `OneOf` unions (type surface explodes across both layers), and third-party result libraries (would tie the public contract to a dependency outside the deliberately minimal `Directory.Packages.props`). Pre-flight checks are kept rather than relying solely on caught constraint exceptions, which would surface a generic message and recreate "branch on index name" coupling. Bare `TypedResults.NotFound()`/`Conflict()` are rejected because the OpenAPI spec would then advertise a bodyless response while the wire carries `ProblemDetails`, dropping the `code` and entity/id fields. Accepted cost: union signatures are slightly over-broad (use `.ToOkOrNotFound<T>()` for read-only endpoints), and a new `ServiceError` variant touches the central `Map` switch — both bounded to single files.
