# Form controls: shadcn/ui vendored components on Base UI primitives

Supplements ADR 0005. The SPA's form controls (select, combobox, date picker, dialog, input, radio, toast, pagination) move from hand-rolled/native implementations to shadcn/ui — vendored component source under `src/components/ui/` — initialised with **Base UI** primitives (`npx shadcn@latest init --base-ui`), not the default Radix. The conversion is a deliberate big-bang: every control converts in one branch (including `Modal` → Dialog and `Toast` → sonner) so the app never lives in a two-dialect state. The hand-rolled `Combobox` is the riskiest swap (grouping, "none"/"create" sentinels, portal min-width feed `AccountSelect` across the categorisation screens), so a parity spike on Base UI's Combobox happens first inside that branch.

The driving problem was interaction-pattern inconsistency, not styling drift: the custom `Combobox`, native `<select>` (`CurrencySelect`), and native date picker (`DateField`) each looked and behaved differently, and per-form helpers (`TextField`/`RadioOption` local to `BankAccountForm`) were re-invented instead of shared. Base UI is chosen over Radix because Radix has been in maintenance mode since the WorkOS acquisition while Base UI (1.0, Dec 2025, by the Radix/Floating UI/MUI authors) is actively developed; shadcn/ui exposes the same component API over both, and this repo has no legacy Radix code to preserve. A headless-only library (React Aria Components) and full styled libraries (Mantine/MUI) were rejected: the former leaves every skin hand-written, the latter fights the ADR 0005 token system. The Balance `@theme` tokens remain the design source of truth — shadcn's expected token names (`--color-background`, `--color-primary`, `--color-ring`, …) are defined as aliases onto Balance tokens in `index.css`, so vendored files stay byte-close to upstream and `shadcn add` keeps working; the shadcn vocabulary is allowed only inside `src/components/ui/`. Consequences: `clsx` + `tailwind-merge` enter the dependency tree (`cn()` replaces the hand-rolled `cx()` — merge semantics are load-bearing for className overrides on vendored components); shadcn's react-hook-form `<Form>` wrapper is explicitly **not** adopted — form state stays `useState` + state modules with FluentValidation errors rendered through `FieldError`; `DateField` keeps its ISO `yyyy-mm-dd` string contract with conversion inside the wrapper.
